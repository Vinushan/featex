# UV Installation & Setup Guide

[uv](https://docs.astral.sh/uv/) is a fast, reliable Python package manager and project runner written in Rust.

## Installation

### macOS / Linux

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

### Windows

```powershell
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
```

### Using Homebrew (macOS)

```bash
brew install uv
```

### Verify Installation

```bash
uv --version
# Output: uv 0.x.y
```

## Quick Start with FeatEx

Once UV is installed, set up the project:

```bash
# Clone and navigate
git clone https://github.com/Vinushan/featex.git
cd featex

# Create virtual environment and install all dependencies
uv sync

# Run tests
uv run pytest

# Use Makefile shortcuts (automatically use uv)
make test
make lint
make format
```

## Common UV Commands

```bash
# Sync dependencies (creates/updates virtual environment)
uv sync

# Run a command in the project environment
uv run pytest
uv run black .

# Run Python scripts
uv run python script.py

# Add a new dependency
uv add package_name

# Remove a dependency
uv remove package_name

# Update dependencies
uv lock --upgrade

# Show dependencies
uv pip list
```

## Why UV?

- ⚡ **10-100x faster** than pip
- 🔒 **Deterministic** - `uv.lock` ensures reproducible environments
- 🎯 **Compatible** - works with existing Python packaging
- 🛠️ **Simpler** - fewer commands, better defaults
- 📦 **Single tool** - replaces pip, pip-tools, virtualenv, and more

## Why UV

| Task | Legacy tool | uv |
|------|-------------|-----|
| Install package | legacy package installer | `uv pip install pkg` |
| Sync dependencies | multiple commands | `uv sync` |
| Run scripts | virtualenv activation | `uv run script.py` |
| Speed | slower | 10-100x faster |
| Reproducibility | manual lock files | built-in with `uv.lock` |

## Working with uv.lock

The `uv.lock` file is automatically created and updated by uv. It ensures everyone on the team has the exact same dependency versions:

```bash
# This file is generated by:
uv sync

# Commit it to version control
git add uv.lock
git commit -m "Update dependencies"

# Other developers run:
git pull
uv sync  # Recreates the exact environment from uv.lock
```

## Troubleshooting

### "uv: command not found"

Make sure UV is in your PATH:

```bash
# Check installation
uv --version

# If not found, add to PATH (check docs for your shell)
export PATH="/home/username/.cargo/bin:$PATH"
```

### Slow first sync?

First `uv sync` might be slow as it downloads and compiles packages. Subsequent syncs are much faster.

### Python version issues?

Create a `.python-version` file (already included) to specify Python version:

```
3.14
```

Then `uv sync` will use or download that version.

### Switching to UV

```bash
uv pip install pkg_name  # Install with uv

# Eventually migrate everything to uv
rm -rf venv/
uv sync
```
